    All-InOne Frontend + Auditoría SDLC¶

    React Vite TypeScript GrapesJS Playwright Auditoría SDLC

    Bienvenido a la documentación del frontend de All-InOne y a la documentación formal de su Auditoría SDLC.

    Desde esta portada puedes elegir entre revisar la documentación técnica del frontend o la documentación de auditoría basada en Project Charter, checklist, evidencias, riesgos y criterios de evaluación.

    • Documentación del Frontend

      React, Vite, TypeScript, estructura de src, rutas protegidas, consumo de API, GrapesJS, componentes, pruebas, seguridad frontend y despliegue de prueba.

      Entrar a Frontend

    • Auditoría SDLC

      Project Charter, criterios de auditoría, metodología, entregables, cronograma, presupuesto, riesgos, checklist SDLC, trazabilidad y defensa en exposición.

      Entrar a Auditoría SDLC

    ¿Por qué están juntas estas dos secciones?¶

    La documentación frontend responde cómo se construye la experiencia visual y la integración con la API.
    La documentación de auditoría responde cómo se evaluó el ciclo de vida del desarrollo del proyecto All-InOne.

    Lectura correcta del sitio: el frontend muestra evidencia técnica de interfaz, rutas, componentes y pruebas; la auditoría SDLC organiza esa evidencia dentro de una evaluación formal.
