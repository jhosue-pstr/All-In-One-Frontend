# Limitaciones

Esta sección registra limitaciones relevantes del frontend dentro del contexto académico del proyecto All-InOne.

## Limitaciones principales

| Limitación | Descripción |
|---|---|
| Entorno académico | El proyecto fue desarrollado como práctica académica, no como sistema en operación formal empresarial. |
| Despliegue de prueba | Puede existir despliegue técnico, pero no implementación oficial para una empresa real. |
| Dependencia del backend | Varias pantallas requieren backend activo y correctamente configurado. |
| Validación de seguridad | El frontend aplica controles visuales, pero la seguridad real depende del backend. |
| Manuales operativos | No se evidencia manual formal de usuario final u operación productiva. |
| Gestión de cambios formal | No se evidencia flujo formal de cambios aprobado para producción. |
| Mantenimiento formal | Las correcciones corresponden al desarrollo/práctica, no a mantenimiento post-producción. |
| Homologación | No se evidencia homologación formal de herramientas, compiladores o tecnologías. |

## Cómo interpretar estas limitaciones

Estas limitaciones no significan que el frontend esté mal desarrollado. Significan que el alcance del proyecto corresponde a una construcción académica con evidencias técnicas, no a una operación empresarial formal.

## Diferencia importante

```text
Despliegue técnico de prueba ≠ Implementación productiva formal
```

## Uso en el checklist SDLC

Por esta razón, los ítems relacionados con implementación formal, post-implementación y mantenimiento productivo deben marcarse con cautela o como no cumplidos cuando exijan evidencia empresarial formal.
