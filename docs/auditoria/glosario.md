# Glosario de auditoría SDLC

| Término | Significado |
|---|---|
| SDLC | Ciclo de Vida del Desarrollo de Software. Incluye planificación, requisitos, diseño, construcción, pruebas, implementación y mantenimiento. |
| Project Charter | Documento de inicio que formaliza la auditoría y define objetivos, alcance, criterios y entregables. |
| Checklist SDLC | Lista de verificación usada para evaluar evidencias del ciclo de vida. |
| Evidencia | Documento, reporte, código, captura, prueba o registro que sustenta una conclusión. |
| Hallazgo | Resultado de auditoría que identifica una conformidad, brecha, riesgo u oportunidad de mejora. |
| No conformidad | Incumplimiento frente a un criterio definido. |
| Trazabilidad | Relación entre requisitos, tareas, módulos, código, pruebas y evidencias. |
| Monolito modular | Arquitectura donde el sistema se despliega como una sola aplicación, pero se organiza internamente por módulos. |
| Multitenancy | Capacidad de atender varios sitios o tenants manteniendo separación de datos y configuración. |
| Soft delete | Eliminación lógica de registros para evitar borrado físico directo. |
| Auditoría | Registro de operaciones relevantes para trazabilidad. |
| DevSecOps | Integración de desarrollo, seguridad y operaciones mediante herramientas y prácticas automatizadas. |
| CI/CD | Integración y despliegue continuo. En este proyecto se evidencia como pipeline técnico/académico, no necesariamente producción formal. |
| Producción formal | Entorno real de operación para usuarios o empresa, con procedimientos controlados de cambio y despliegue. |
| Evidencia equivalente | Capturas, videos, reportes o registros usados cuando no existe acceso permanente a un entorno funcional. |
