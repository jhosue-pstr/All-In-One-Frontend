# Project Charter de Auditoría SDLC

<div class="audit-callout" markdown>

**Propósito:** formalizar la auditoría del Ciclo de Vida del Desarrollo de Software del proyecto **All-InOne** y establecer la base para evaluar su gestión, requisitos, arquitectura, construcción, pruebas, seguridad, documentación, implementación y mantenimiento.

</div>

## ¿Por qué se incluye en MkDocs?

El Project Charter no es solo un documento administrativo. Dentro del MkDocs cumple tres funciones importantes:

1. **Contextualiza el sistema auditado:** explica qué es All-InOne, qué problema atiende y qué módulos contiene.
2. **Delimita la auditoría:** define qué se revisa, qué no se revisa y bajo qué criterios se evalúa.
3. **Sustenta la exposición:** permite mostrar que la revisión no se hizo de forma improvisada, sino con objetivos, alcance, metodología y entregables.

## Valor para la auditoría

El Project Charter permite que el checklist y los hallazgos tengan una base clara. Por ejemplo, si el checklist evalúa pruebas, seguridad o mantenimiento, el Project Charter ya establece que esos aspectos están dentro del alcance.

!!! tip "Cómo explicarlo"
    “El Project Charter funciona como el documento de inicio y control de la auditoría. Define qué se va a evaluar, con qué criterios, en qué periodo, con qué entregables y bajo qué restricciones.”

## Relación con el sistema

All-InOne se describe como una plataforma SaaS multitenant con módulos como Sitios, Plantillas, Blog, Tienda, Auth Público, Analítica, auditoría y soft delete. Esta complejidad justifica que la revisión SDLC no se limite a verificar si el sistema corre, sino también si existe coherencia entre documentación, arquitectura, código, pruebas y evidencias.
